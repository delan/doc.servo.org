21/// Defines a source that should be used to deserialize a value in the next call
22/// to [`next_value_seed()`](MapAccess::next_value_seed)
23#[derive(Debug, PartialEq)]
24enum ValueSource {
25    /// Source are not specified, because [`next_key_seed()`] not yet called.
26    /// This is an initial state and state after deserializing value
27    /// (after call of [`next_value_seed()`]).
28    ///
29    /// Attempt to call [`next_value_seed()`] while accessor in this state would
30    /// return a [`DeError::KeyNotRead`] error.
31    ///
32    /// [`next_key_seed()`]: MapAccess::next_key_seed
33    /// [`next_value_seed()`]: MapAccess::next_value_seed
34    Unknown,
35    /// Next value should be deserialized from an attribute value; value is located
36    /// at specified span.
37    Attribute(Range<usize>),
38    /// Value should be deserialized from the text content of the XML node, which
39    /// represented or by an ordinary text node, or by a CDATA node:
40    ///
41    /// ```xml
42    /// <any-tag>
43    ///     <key>text content</key>
44    /// <!--     ^^^^^^^^^^^^ - this will be used to deserialize map value -->
45    /// </any-tag>
46    /// ```
47    /// ```xml
48    /// <any-tag>
49    ///     <key><![CDATA[cdata content]]></key>
50    /// <!--              ^^^^^^^^^^^^^ - this will be used to deserialize a map value -->
51    /// </any-tag>
52    /// ```
53    Text,
54    /// Next value should be deserialized from an element with an any name, except
55    /// elements with a name matching one of the struct fields. Corresponding tag
56    /// name will always be associated with a field with name [`VALUE_KEY`].
57    ///
58    /// That state is set when call to [`peek()`] returns a [`Start`] event, which
59    /// [`name()`] is not listed in the [list of known fields] (which for a struct
60    /// is a list of field names, and for a map that is an empty list), _and_
61    /// struct has a field with a special name [`VALUE_KEY`].
62    ///
63    /// When in this state, next event, returned by [`next()`], will be a [`Start`],
64    /// which represents both a key, and a value. Value would be deserialized from
65    /// the whole element and how is will be done determined by the value deserializer.
66    /// The [`ElementMapAccess`] do not consume any events in that state.
67    ///
68    /// Because in that state any encountered `<tag>` is mapped to the [`VALUE_KEY`]
69    /// field, it is possible to use tag name as an enum discriminator, so `enum`s
70    /// can be deserialized from that XMLs:
71    ///
72    /// ```xml
73    /// <any-tag>
74    ///     <variant1>...</variant1>
75    /// <!-- ~~~~~~~~               - this data will determine that this is Enum::variant1 -->
76    /// <!--^^^^^^^^^^^^^^^^^^^^^^^ - this data will be used to deserialize a map value -->
77    /// </any-tag>
78    /// ```
79    /// ```xml
80    /// <any-tag>
81    ///     <variant2>...</variant2>
82    /// <!-- ~~~~~~~~               - this data will determine that this is Enum::variant2 -->
83    /// <!--^^^^^^^^^^^^^^^^^^^^^^^ - this data will be used to deserialize a map value -->
84    /// </any-tag>
85    /// ```
86    ///
87    /// both can be deserialized into
88    ///
89    /// ```ignore
90    /// enum Enum {
91    ///   variant1,
92    ///   variant2,
93    /// }
94    /// struct AnyName {
95    ///   #[serde(rename = "$value")]
96    ///   field: Enum,
97    /// }
98    /// ```
99    ///
100    /// That is possible, because value deserializer have access to the full content
101    /// of a `<variant1>...</variant1>` or `<variant2>...</variant2>` node, including
102    /// the tag name.
103    ///
104    /// [`Start`]: DeEvent::Start
105    /// [`peek()`]: Deserializer::peek()
106    /// [`next()`]: Deserializer::next()
107    /// [`name()`]: BytesStart::name()
108    /// [`Text`]: Self::Text
109    /// [list of known fields]: ElementMapAccess::fields
110    Content,
111    /// Next value should be deserialized from an element with a dedicated name.
112    /// If deserialized type is a sequence, then that sequence will collect all
113    /// elements with the same name until it will be filled. If not all elements
114    /// would be consumed, the rest will be ignored.
115    ///
116    /// That state is set when call to [`peek()`] returns a [`Start`] event, which
117    /// [`name()`] represents a field name. That name will be deserialized as a key.
118    ///
119    /// When in this state, next event, returned by [`next()`], will be a [`Start`],
120    /// which represents both a key, and a value. Value would be deserialized from
121    /// the whole element and how is will be done determined by the value deserializer.
122    /// The [`ElementMapAccess`] do not consume any events in that state.
123    ///
124    /// An illustration below shows, what data is used to deserialize key and value:
125    /// ```xml
126    /// <any-tag>
127    ///     <key>...</key>
128    /// <!-- ~~~           - this data will be used to deserialize a map key -->
129    /// <!--^^^^^^^^^^^^^^ - this data will be used to deserialize a map value -->
130    /// </any-tag>
131    /// ```
132    ///
133    /// Although value deserializer will have access to the full content of a `<key>`
134    /// node (including the tag name), it will not get much benefits from that,
135    /// because tag name will always be fixed for a given map field (equal to a
136    /// field name). So, if the field type is an `enum`, it cannot select its
137    /// variant based on the tag name. If that is needed, then [`Content`] variant
138    /// of this enum should be used. Such usage is enabled by annotating a struct
139    /// field as "content" field, which implemented as given the field a special
140    /// [`VALUE_KEY`] name.
141    ///
142    /// [`Start`]: DeEvent::Start
143    /// [`peek()`]: Deserializer::peek()
144    /// [`next()`]: Deserializer::next()
145    /// [`name()`]: BytesStart::name()
146    /// [`Content`]: Self::Content
147    Nested,
148}

152/// A deserializer that extracts map-like structures from an XML. This deserializer
153/// represents a one XML tag:
154///
155/// ```xml
156/// <tag>...</tag>
157/// ```
158///
159/// Name of this tag is stored in a [`Self::start`] property.
160///
161/// # Lifetimes
162///
163/// - `'de` lifetime represents a buffer, from which deserialized values can
164///   borrow their data. Depending on the underlying reader, there can be an
165///   internal buffer of deserializer (i.e. deserializer itself) or an input
166///   (in that case it is possible to approach zero-copy deserialization).
167///
168/// - `'d` lifetime represents a parent deserializer, which could own the data
169///   buffer.
170pub(crate) struct ElementMapAccess<'de, 'd, R, E>
171where
172    R: XmlRead<'de>,
173    E: EntityResolver,
174{
175    /// Tag -- owner of attributes
176    start: BytesStart<'de>,
177    de: &'d mut Deserializer<'de, R, E>,
178    /// State of the iterator over attributes. Contains the next position in the
179    /// inner `start` slice, from which next attribute should be parsed.
180    iter: IterState,
181    /// Current state of the accessor that determines what next call to API
182    /// methods should return.
183    source: ValueSource,
184    /// List of field names of the struct. It is empty for maps
185    fields: &'static [&'static str],
186    /// If `true`, then the deserialized struct has a field with a special name:
187    /// [`VALUE_KEY`]. That field should be deserialized from the whole content
188    /// of an XML node, including tag name:
189    ///
190    /// ```xml
191    /// <tag>value for VALUE_KEY field<tag>
192    /// ```
193    has_value_field: bool,
194    /// If `true`, then the deserialized struct has a field with a special name:
195    /// [`TEXT_KEY`].
196    has_text_field: bool,
197}
198
199impl<'de, 'd, R, E> ElementMapAccess<'de, 'd, R, E>
200where
201    R: XmlRead<'de>,
202    E: EntityResolver,
203{
204    /// Create a new ElementMapAccess
205    pub fn new(
206        de: &'d mut Deserializer<'de, R, E>,
207        start: BytesStart<'de>,
208        fields: &'static [&'static str],
209    ) -> Self {
210        Self {
211            de,
212            iter: IterState::new(start.name().as_ref().len(), false),
213            start,
214            source: ValueSource::Unknown,
215            fields,
216            has_value_field: fields.contains(&VALUE_KEY),
217            has_text_field: fields.contains(&TEXT_KEY),
218        }
219    }
220
221    /// Determines if subtree started with the specified event shoould be skipped.
222    ///
223    /// Used to map elements with `xsi:nil` attribute set to true to `None` in optional contexts.
224    ///
225    /// We need to handle two attributes:
226    /// - on parent element: `<map xsi:nil="true"><foo/></map>`
227    /// - on this element:   `<map><foo xsi:nil="true"/></map>`
228    ///
229    /// We check parent element too because `xsi:nil` affects only nested elements of the
230    /// tag where it is defined. We can map structure with fields mapped to attributes to
231    /// the `<map>` element and set to `None` all its optional elements.
232    fn should_skip_subtree(&self, start: &BytesStart) -> bool {
233        self.de.reader.reader.has_nil_attr(&self.start) || self.de.reader.reader.has_nil_attr(start)
234    }
235
236    /// Skips whitespaces when they are not preserved
237    #[inline]
238    fn skip_whitespaces(&mut self) -> Result<(), DeError> {
239        // TODO: respect the `xml:space` attribute and probably some deserialized type sign
240        self.de.skip_whitespaces()
241    }
242}

244impl<'de, 'd, R, E> MapAccess<'de> for ElementMapAccess<'de, 'd, R, E>
245where
246    R: XmlRead<'de>,
247    E: EntityResolver,
248{
249    type Error = DeError;
250
251    fn next_key_seed<K: DeserializeSeed<'de>>(
252        &mut self,
253        seed: K,
254    ) -> Result<Option<K::Value>, Self::Error> {
255        debug_assert_eq!(self.source, ValueSource::Unknown);
256
257        // FIXME: There error positions counted from the start of tag name - need global position
258        let slice = &self.start.buf;
259        let decoder = self.start.decoder();
260
261        if let Some(a) = self.iter.next(slice).transpose()? {
262            // try getting map from attributes (key= "value")
263            let (key, value) = a.into();
264            self.source = ValueSource::Attribute(value.unwrap_or_default());
265
266            // Attributes in mapping starts from @ prefix
267            // TODO: Customization point - may customize prefix
268            self.de.key_buf.clear();
269            self.de.key_buf.push('@');
270
271            let de =
272                QNameDeserializer::from_attr(QName(&slice[key]), decoder, &mut self.de.key_buf)?;
273            seed.deserialize(de).map(Some)
274        } else {
275            self.skip_whitespaces()?;
276            // try getting from events (<key>value</key>)
277            match self.de.peek()? {
278                // If we have dedicated "$text" field, it will not be passed to "$value" field
279                DeEvent::Text(_) if self.has_value_field && !self.has_text_field => {
280                    self.source = ValueSource::Content;
281                    // Deserialize `key` from special attribute name which means
282                    // that value should be taken from the text content of the
283                    // XML node
284                    let de = BorrowedStrDeserializer::<DeError>::new(VALUE_KEY);
285                    seed.deserialize(de).map(Some)
286                }
287                DeEvent::Text(_) => {
288                    self.source = ValueSource::Text;
289                    // Deserialize `key` from special attribute name which means
290                    // that value should be taken from the text content of the
291                    // XML node
292                    let de = BorrowedStrDeserializer::<DeError>::new(TEXT_KEY);
293                    seed.deserialize(de).map(Some)
294                }
295                // Used to deserialize collections of enums, like:
296                // <root>
297                //   <A/>
298                //   <B/>
299                //   <C/>
300                // </root>
301                //
302                // into
303                //
304                // enum Enum { A, B, ะก }
305                // struct Root {
306                //     #[serde(rename = "$value")]
307                //     items: Vec<Enum>,
308                // }
309                // TODO: This should be handled by #[serde(flatten)]
310                // See https://github.com/serde-rs/serde/issues/1905
311                DeEvent::Start(e) if self.has_value_field && not_in(self.fields, e)? => {
312                    self.source = ValueSource::Content;
313
314                    let de = BorrowedStrDeserializer::<DeError>::new(VALUE_KEY);
315                    seed.deserialize(de).map(Some)
316                }
317                DeEvent::Start(e) => {
318                    self.source = ValueSource::Nested;
319
320                    let de = QNameDeserializer::from_elem(e)?;
321                    seed.deserialize(de).map(Some)
322                }
323                // Stop iteration after reaching a closing tag
324                // The matching tag name is guaranteed by the reader if our
325                // deserializer implementation is correct
326                DeEvent::End(e) => {
327                    debug_assert_eq!(self.start.name(), e.name());
328                    // Consume End
329                    self.de.next()?;
330                    Ok(None)
331                }
332                // We cannot get `Eof` legally, because we always inside of the
333                // opened tag `self.start`
334                DeEvent::Eof => {
335                    Err(Error::missed_end(self.start.name(), self.start.decoder()).into())
336                }
337            }
338        }
339    }
340
341    fn next_value_seed<K: DeserializeSeed<'de>>(
342        &mut self,
343        seed: K,
344    ) -> Result<K::Value, Self::Error> {
345        match std::mem::replace(&mut self.source, ValueSource::Unknown) {
346            ValueSource::Attribute(value) => seed.deserialize(SimpleTypeDeserializer::from_part(
347                &self.start.buf,
348                value,
349                self.start.decoder(),
350            )),
351            // This arm processes the following XML shape:
352            // <any-tag>
353            //   text value
354            // </any-tag>
355            // The whole map represented by an `<any-tag>` element, the map key
356            // is implicit and equals to the `TEXT_KEY` constant, and the value
357            // is a `Text` event (the value deserializer will see that event)
358            // This case are checked by "xml_schema_lists::element" tests in tests/serde-de.rs
359            ValueSource::Text => match self.de.next()? {
360                DeEvent::Text(e) => seed.deserialize(SimpleTypeDeserializer::from_text_content(e)),
361                // SAFETY: We set `Text` only when we seen `Text`
362                _ => unreachable!(),
363            },
364            // This arm processes the following XML shape:
365            // <any-tag>
366            //   <any>...</any>
367            // </any-tag>
368            // The whole map represented by an `<any-tag>` element, the map key
369            // is implicit and equals to the `VALUE_KEY` constant, and the value
370            // is a `Start` event (the value deserializer will see that event)
371            ValueSource::Content => seed.deserialize(MapValueDeserializer {
372                map: self,
373                fixed_name: false,
374            }),
375            // This arm processes the following XML shape:
376            // <any-tag>
377            //   <tag>...</tag>
378            // </any-tag>
379            // The whole map represented by an `<any-tag>` element, the map key
380            // is a `tag`, and the value is a `Start` event (the value deserializer
381            // will see that event)
382            ValueSource::Nested => seed.deserialize(MapValueDeserializer {
383                map: self,
384                fixed_name: true,
385            }),
386            ValueSource::Unknown => Err(DeError::KeyNotRead),
387        }
388    }
389}

393/// A deserializer for a value of map or struct. That deserializer slightly
394/// differently processes events for a primitive types and sequences than
395/// a [`Deserializer`].
396///
397/// This deserializer used to deserialize two kinds of fields:
398/// - usual fields with a dedicated name, such as `field_one` or `field_two`, in
399///   that case field [`Self::fixed_name`] is `true`;
400/// - the special `$value` field which represents any tag or a textual content
401///   in the XML which would be found in the document, in that case field
402///   [`Self::fixed_name`] is `false`.
403///
404/// This deserializer can see two kind of events at the start:
405/// - [`DeEvent::Text`]
406/// - [`DeEvent::Start`]
407///
408/// which represents two possible variants of items:
409/// ```xml
410/// <item>A tag item</item>
411/// A text item
412/// <yet another="tag item"/>
413/// ```
414///
415/// This deserializer are very similar to a [`ElementDeserializer`]. The only difference
416/// in the `deserialize_seq` method. This deserializer will act as an iterator
417/// over tags / text within it's parent tag, whereas the [`ElementDeserializer`]
418/// will represent sequences as an `xs:list`.
419///
420/// This deserializer processes items as following:
421/// - primitives (numbers, booleans, strings, characters) are deserialized either
422///   from a text content, or unwrapped from a one level of a tag. So, `123` and
423///   `<int>123</int>` both can be deserialized into an `u32`;
424/// - `Option`:
425///   - empty text of [`DeEvent::Text`] is deserialized as `None`;
426///   - everything else are deserialized as `Some` using the same deserializer,
427///     including `<tag/>` or `<tag></tag>`;
428/// - units (`()`) and unit structs consumes the whole text or element subtree;
429/// - newtype structs are deserialized by forwarding deserialization of inner type
430///   with the same deserializer;
431/// - sequences, tuples and tuple structs are deserialized by iterating within the
432///   parent tag and deserializing each tag or text content using [`ElementDeserializer`];
433/// - structs and maps are deserialized using new instance of [`ElementMapAccess`];
434/// - enums:
435///   - in case of [`DeEvent::Text`] event the text content is deserialized as
436///     a `$text` variant. Enum content is deserialized from the text using
437///     [`SimpleTypeDeserializer`];
438///   - in case of [`DeEvent::Start`] event the tag name is deserialized as
439///     an enum tag, and the content inside are deserialized as an enum content.
440///     Depending on a variant kind deserialization is performed as:
441///     - unit variants: consuming text content or a subtree;
442///     - newtype variants: forward deserialization to the inner type using
443///       this deserializer;
444///     - tuple variants: call [`deserialize_tuple`] of this deserializer;
445///     - struct variants: call [`deserialize_struct`] of this deserializer.
446///
447/// [`deserialize_tuple`]: #method.deserialize_tuple
448/// [`deserialize_struct`]: #method.deserialize_struct
449struct MapValueDeserializer<'de, 'd, 'm, R, E>
450where
451    R: XmlRead<'de>,
452    E: EntityResolver,
453{
454    /// Access to the map that created this deserializer. Gives access to the
455    /// context, such as list of fields, that current map known about.
456    map: &'m mut ElementMapAccess<'de, 'd, R, E>,
457    /// Whether this deserializer was created for deserialization from an element
458    /// with fixed name, or the elements with different names or even text are allowed.
459    ///
460    /// If this field is `true`, we process `<tag>` element in the following XML shape:
461    ///
462    /// ```xml
463    /// <any-tag>
464    ///   <tag>...</tag>
465    /// </any-tag>
466    /// ```
467    ///
468    /// The whole map represented by an `<any-tag>` element, the map key is a `tag`,
469    /// and the value starts with is a `Start("tag")` (the value deserializer will
470    /// see that event first) and extended to the matching `End("tag")` event.
471    /// In order to deserialize primitives (such as `usize`) we need to allow to
472    /// look inside the one levels of tags, so the
473    ///
474    /// ```xml
475    /// <tag>42<tag>
476    /// ```
477    ///
478    /// could be deserialized into `42usize` without problems, and at the same time
479    ///
480    /// ```xml
481    /// <tag>
482    ///   <key1/>
483    ///   <key2/>
484    ///   <!--...-->
485    /// <tag>
486    /// ```
487    /// could be deserialized to a struct.
488    ///
489    /// If this field is `false`, we processes the one of following XML shapes:
490    ///
491    /// ```xml
492    /// <any-tag>
493    ///   text value
494    /// </any-tag>
495    /// ```
496    /// ```xml
497    /// <any-tag>
498    ///   <![CDATA[cdata value]]>
499    /// </any-tag>
500    /// ```
501    /// ```xml
502    /// <any-tag>
503    ///   <any>...</any>
504    /// </any-tag>
505    /// ```
506    ///
507    /// The whole map represented by an `<any-tag>` element, the map key is
508    /// implicit and equals to the [`VALUE_KEY`] constant, and the value is
509    /// a [`Text`], or a [`Start`] event (the value deserializer will see one of
510    /// those events). In the first two cases the value of this field do not matter
511    /// (because we already see the textual event and there no reasons to look
512    /// "inside" something), but in the last case the primitives should raise
513    /// a deserialization error, because that means that you trying to deserialize
514    /// the following struct:
515    ///
516    /// ```ignore
517    /// struct AnyName {
518    ///   #[serde(rename = "$value")]
519    ///   any_name: String,
520    /// }
521    /// ```
522    /// which means that `any_name` should get a content of the `<any-tag>` element.
523    ///
524    /// Changing this can be valuable for <https://github.com/tafia/quick-xml/issues/383>,
525    /// but those fields should be explicitly marked that they want to get any
526    /// possible markup as a `String` and that mark is different from marking them
527    /// as accepting "text content" which the currently `$text` means.
528    ///
529    /// [`Text`]: DeEvent::Text
530    /// [`Start`]: DeEvent::Start
531    fixed_name: bool,
532}
533
534impl<'de, 'd, 'm, R, E> MapValueDeserializer<'de, 'd, 'm, R, E>
535where
536    R: XmlRead<'de>,
537    E: EntityResolver,

539    /// Returns a next string as concatenated content of consequent [`Text`] and
540    /// [`CData`] events, used inside [`deserialize_primitives!()`].
541    ///
542    /// [`Text`]: crate::events::Event::Text
543    /// [`CData`]: crate::events::Event::CData
544    #[inline]

547        self.map.de.read_string_impl(self.fixed_name)
548    }

551impl<'de, 'd, 'm, R, E> de::Deserializer<'de> for MapValueDeserializer<'de, 'd, 'm, R, E>
552where
553    R: XmlRead<'de>,
554    E: EntityResolver,
555{
556    type Error = DeError;
557
558    deserialize_primitives!(mut);
559
560    #[inline]
561    fn deserialize_unit<V>(self, visitor: V) -> Result<V::Value, Self::Error>
562    where
563        V: Visitor<'de>,
564    {
565        self.map.de.deserialize_unit(visitor)
566    }
567
568    fn deserialize_option<V>(self, visitor: V) -> Result<V::Value, Self::Error>
569    where
570        V: Visitor<'de>,
571    {
572        // We cannot use result of `peek()` directly because of borrow checker
573        let _ = self.map.de.peek()?;
574        match self.map.de.last_peeked() {
575            DeEvent::Text(t) if t.is_empty() => visitor.visit_none(),
576            DeEvent::Start(start) if self.map.should_skip_subtree(start) => {
577                self.map.de.skip_next_tree()?;
578                visitor.visit_none()
579            }
580            _ => visitor.visit_some(self),
581        }
582    }
583
584    /// Forwards deserialization of the inner type. Always calls [`Visitor::visit_newtype_struct`]
585    /// with the same deserializer.
586    fn deserialize_newtype_struct<V>(
587        self,
588        _name: &'static str,
589        visitor: V,
590    ) -> Result<V::Value, Self::Error>
591    where
592        V: Visitor<'de>,
593    {
594        visitor.visit_newtype_struct(self)
595    }
596
597    /// Deserializes each `<tag>` in
598    /// ```xml
599    /// <any-tag>
600    ///   <tag>...</tag>
601    ///   <tag>...</tag>
602    ///   <tag>...</tag>
603    /// </any-tag>
604    /// ```
605    /// as a sequence item, where `<any-tag>` represents a Map in a [`Self::map`],
606    /// and a `<tag>` is a sequential field of that map.
607    fn deserialize_seq<V>(self, visitor: V) -> Result<V::Value, Self::Error>
608    where
609        V: Visitor<'de>,
610    {
611        let filter = if self.fixed_name {
612            match self.map.de.peek()? {
613                // Clone is cheap if event borrows from the input
614                DeEvent::Start(e) => TagFilter::Include(e.clone()),
615                // SAFETY: we use that deserializer with `fixed_name == true`
616                // only from the `ElementMapAccess::next_value_seed` and only when we
617                // peeked `Start` event
618                _ => unreachable!(),
619            }
620        } else {
621            TagFilter::Exclude(self.map.fields, self.map.has_text_field)
622        };
623        visitor.visit_seq(MapValueSeqAccess {
624            #[cfg(feature = "overlapped-lists")]
625            checkpoint: self.map.de.skip_checkpoint(),
626
627            map: self.map,
628            filter,
629        })
630    }
631
632    #[inline]
633    fn deserialize_struct<V>(
634        self,
635        name: &'static str,
636        fields: &'static [&'static str],
637        visitor: V,
638    ) -> Result<V::Value, Self::Error>
639    where
640        V: Visitor<'de>,
641    {
642        self.map.de.deserialize_struct(name, fields, visitor)
643    }
644
645    fn deserialize_enum<V>(
646        self,
647        _name: &'static str,
648        _variants: &'static [&'static str],
649        visitor: V,
650    ) -> Result<V::Value, Self::Error>
651    where
652        V: Visitor<'de>,
653    {
654        if self.fixed_name {
655            match self.map.de.next()? {
656                // Handles <field>UnitEnumVariant</field>
657                DeEvent::Start(e) => {
658                    // skip <field>, read text after it and ensure that it is ended by </field>
659                    let text = self.map.de.read_text(e.name())?;
660                    if text.is_empty() {
661                        // Map empty text (<field/>) to a special `$text` variant
662                        visitor.visit_enum(SimpleTypeDeserializer::from_text(TEXT_KEY.into()))
663                    } else {
664                        visitor.visit_enum(SimpleTypeDeserializer::from_text(text))
665                    }
666                }
667                // SAFETY: we use that deserializer with `fixed_name == true`
668                // only from the `MapAccess::next_value_seed` and only when we
669                // peeked `Start` event
670                _ => unreachable!(),
671            }
672        } else {
673            visitor.visit_enum(self)
674        }
675    }
676
677    fn deserialize_any<V>(self, visitor: V) -> Result<V::Value, Self::Error>
678    where
679        V: Visitor<'de>,
680    {
681        match self.map.de.peek()? {
682            DeEvent::Text(_) => self.deserialize_str(visitor),
683            _ => self.deserialize_map(visitor),
684        }
685    }
686}

719struct MapValueVariantAccess<'de, 'd, 'm, R, E>
720where
721    R: XmlRead<'de>,
722    E: EntityResolver,
723{
724    /// Access to the map that created this enum accessor. Gives access to the
725    /// context, such as list of fields, that current map known about.
726    map: &'m mut ElementMapAccess<'de, 'd, R, E>,
727    /// `true` if variant should be deserialized from a textual content
728    /// and `false` if from tag
729    is_text: bool,
730}
731
732impl<'de, 'd, 'm, R, E> de::VariantAccess<'de> for MapValueVariantAccess<'de, 'd, 'm, R, E>
733where
734    R: XmlRead<'de>,
735    E: EntityResolver,
736{
737    type Error = DeError;
738
739    fn unit_variant(self) -> Result<(), Self::Error> {
740        match self.map.de.next()? {
741            // Consume subtree
742            DeEvent::Start(e) => self.map.de.read_to_end(e.name()),
743            // Does not needed to deserialize using SimpleTypeDeserializer, because
744            // it returns `()` when `deserialize_unit()` is requested
745            DeEvent::Text(_) => Ok(()),
746            // SAFETY: the other events are filtered in `variant_seed()`
747            _ => unreachable!("Only `Start` or `Text` events are possible here"),
748        }
749    }
750
751    fn newtype_variant_seed<T>(self, seed: T) -> Result<T::Value, Self::Error>
752    where
753        T: DeserializeSeed<'de>,
754    {
755        if self.is_text {
756            match self.map.de.next()? {
757                DeEvent::Text(e) => seed.deserialize(SimpleTypeDeserializer::from_text_content(e)),
758                // SAFETY: the other events are filtered in `variant_seed()`
759                _ => unreachable!("Only `Text` events are possible here"),
760            }
761        } else {
762            seed.deserialize(MapValueDeserializer {
763                map: self.map,
764                // Because element name already was either mapped to a field name,
765                // or to a variant name, we should not treat it as variable
766                fixed_name: true,
767            })
768        }
769    }
770
771    fn tuple_variant<V>(self, len: usize, visitor: V) -> Result<V::Value, Self::Error>
772    where
773        V: Visitor<'de>,
774    {
775        if self.is_text {
776            match self.map.de.next()? {
777                DeEvent::Text(e) => {
778                    SimpleTypeDeserializer::from_text_content(e).deserialize_tuple(len, visitor)
779                }
780                // SAFETY: the other events are filtered in `variant_seed()`
781                _ => unreachable!("Only `Text` events are possible here"),
782            }
783        } else {
784            MapValueDeserializer {
785                map: self.map,
786                // Because element name already was either mapped to a field name,
787                // or to a variant name, we should not treat it as variable
788                fixed_name: true,
789            }
790            .deserialize_tuple(len, visitor)
791        }
792    }
793
794    fn struct_variant<V>(
795        self,
796        fields: &'static [&'static str],
797        visitor: V,
798    ) -> Result<V::Value, Self::Error>
799    where
800        V: Visitor<'de>,
801    {
802        match self.map.de.next()? {
803            DeEvent::Start(e) => visitor.visit_map(ElementMapAccess::new(self.map.de, e, fields)),
804            DeEvent::Text(e) => {
805                SimpleTypeDeserializer::from_text_content(e).deserialize_struct("", fields, visitor)
806            }
807            // SAFETY: the other events are filtered in `variant_seed()`
808            _ => unreachable!("Only `Start` or `Text` events are possible here"),
809        }
810    }
811}

815/// Check if tag `start` is included in the `fields` list. `decoder` is used to
816/// get a string representation of a tag.
817///
818/// Returns `true`, if `start` is not in the `fields` list and `false` otherwise.
819fn not_in(fields: &'static [&'static str], start: &BytesStart) -> Result<bool, DeError> {
820    let tag = start.decoder().decode(start.local_name().into_inner())?;
821
822    Ok(fields.iter().all(|&field| field != tag.as_ref()))
823}

825/// A filter that determines, what tags should form a sequence.
826///
827/// There are two types of sequences:
828/// - sequence where each element represented by tags with the same name
829/// - sequence where each element can have a different tag
830///
831/// The first variant could represent a collection of structs, the second --
832/// a collection of enum variants.
833///
834/// In the second case we don't know what tag name should be expected as a
835/// sequence element, so we accept any element. Since the sequence are flattened
836/// into maps, we skip elements which have dedicated fields in a struct by using an
837/// `Exclude` filter that filters out elements with names matching field names
838/// from the struct.
839///
840/// # Lifetimes
841///
842/// `'de` represents a lifetime of the XML input, when filter stores the
843/// dedicated tag name
844#[derive(Debug)]
845enum TagFilter<'de> {
846    /// A `SeqAccess` interested only in tags with specified name to deserialize
847    /// an XML like this:
848    ///
849    /// ```xml
850    /// <...>
851    ///   <tag/>
852    ///   <tag/>
853    ///   <tag/>
854    ///   ...
855    /// </...>
856    /// ```
857    ///
858    /// The tag name is stored inside (`b"tag"` for that example)
859    Include(BytesStart<'de>), //TODO: Need to store only name instead of a whole tag
860    /// A `SeqAccess` interested in tags with any name, except explicitly listed.
861    /// Excluded tags are used as struct field names and therefore should not
862    /// fall into a `$value` category.
863    ///
864    /// The `bool` represents the having of a `$text` special field in fields array.
865    /// It is used to exclude text events when `$text` fields is defined together with
866    /// `$value` fieldะฑ and `$value` accepts sequence.
867    Exclude(&'static [&'static str], bool),
868}
869
870impl<'de> TagFilter<'de> {
871    fn is_suitable(&self, start: &BytesStart) -> Result<bool, DeError> {
872        match self {
873            Self::Include(n) => Ok(n.name() == start.name()),
874            Self::Exclude(fields, _) => not_in(fields, start),
875        }
876    }
877    const fn need_skip_text(&self) -> bool {
878        match self {
879            // If we look only for tags, we should skip any $text keys
880            Self::Include(_) => true,
881            // If we look fo any data, we should exclude $text keys if it in the list
882            Self::Exclude(_, has_text_field) => *has_text_field,
883        }
884    }
885}

889/// An accessor to sequence elements forming a value for struct field.
890/// Technically, this sequence is flattened out into structure and sequence
891/// elements are overlapped with other fields of a structure. Each call to
892/// [`Self::next_element_seed`] consumes a next sub-tree or consequent list
893/// of [`Text`] and [`CData`] events.
894///
895/// ```xml
896/// <>
897///   ...
898///   <item>The is the one item</item>
899///   This is <![CDATA[one another]]> item<!-- even when--> it splitted by comments
900///   <tag>...and that is the third!</tag>
901///   ...
902/// </>
903/// ```
904///
905/// Depending on [`Self::filter`], only some of that possible constructs would be
906/// an element.
907///
908/// [`Text`]: crate::events::Event::Text
909/// [`CData`]: crate::events::Event::CData
910struct MapValueSeqAccess<'de, 'd, 'm, R, E>
911where
912    R: XmlRead<'de>,
913    E: EntityResolver,
914{
915    /// Accessor to a map that creates this accessor and to a deserializer for
916    /// a sequence items.
917    map: &'m mut ElementMapAccess<'de, 'd, R, E>,
918    /// Filter that determines whether a tag is a part of this sequence.
919    ///
920    /// When feature [`overlapped-lists`] is not activated, iteration will stop
921    /// when found a tag that does not pass this filter.
922    ///
923    /// When feature [`overlapped-lists`] is activated, all tags, that not pass
924    /// this check, will be skipped.
925    ///
926    /// [`overlapped-lists`]: ../../index.html#overlapped-lists
927    filter: TagFilter<'de>,
928
929    /// Checkpoint after which all skipped events should be returned. All events,
930    /// that was skipped before creating this checkpoint, will still stay buffered
931    /// and will not be returned
932    #[cfg(feature = "overlapped-lists")]
933    checkpoint: usize,
934}
935
936#[cfg(feature = "overlapped-lists")]
937impl<'de, 'd, 'm, R, E> Drop for MapValueSeqAccess<'de, 'd, 'm, R, E>
938where
939    R: XmlRead<'de>,
940    E: EntityResolver,
941{
942    fn drop(&mut self) {
943        self.map.de.start_replay(self.checkpoint);
944    }
945}

947impl<'de, 'd, 'm, R, E> SeqAccess<'de> for MapValueSeqAccess<'de, 'd, 'm, R, E>
948where
949    R: XmlRead<'de>,
950    E: EntityResolver,
951{
952    type Error = DeError;
953
954    fn next_element_seed<T>(&mut self, seed: T) -> Result<Option<T::Value>, DeError>
955    where
956        T: DeserializeSeed<'de>,
957    {
958        loop {
959            self.map.skip_whitespaces()?;
960            break match self.map.de.peek()? {
961                // If we see a tag that we not interested, skip it
962                #[cfg(feature = "overlapped-lists")]
963                DeEvent::Start(e) if !self.filter.is_suitable(e)? => {
964                    self.map.de.skip()?;
965                    continue;
966                }
967                // Skip any text events if sequence expects only specific tag names
968                #[cfg(feature = "overlapped-lists")]
969                DeEvent::Text(_) if self.filter.need_skip_text() => {
970                    self.map.de.skip()?;
971                    continue;
972                }
973                // Stop iteration when list elements ends
974                #[cfg(not(feature = "overlapped-lists"))]
975                DeEvent::Start(e) if !self.filter.is_suitable(e)? => Ok(None),
976                #[cfg(not(feature = "overlapped-lists"))]
977                DeEvent::Text(_) if self.filter.need_skip_text() => Ok(None),
978
979                // Stop iteration after reaching a closing tag
980                // The matching tag name is guaranteed by the reader
981                DeEvent::End(e) => {
982                    debug_assert_eq!(self.map.start.name(), e.name());
983                    Ok(None)
984                }
985                // We cannot get `Eof` legally, because we always inside of the
986                // opened tag `self.map.start`
987                DeEvent::Eof => {
988                    Err(Error::missed_end(self.map.start.name(), self.map.start.decoder()).into())
989                }
990
991                DeEvent::Text(_) => match self.map.de.next()? {
992                    DeEvent::Text(e) => seed.deserialize(TextDeserializer(e)).map(Some),
993                    // SAFETY: we just checked that the next event is Text
994                    _ => unreachable!(),
995                },
996                DeEvent::Start(_) => match self.map.de.next()? {
997                    DeEvent::Start(start) => seed
998                        .deserialize(ElementDeserializer {
999                            start,
1000                            de: self.map.de,
1001                        })
1002                        .map(Some),
1003                    // SAFETY: we just checked that the next event is Start
1004                    _ => unreachable!(),
1005                },
1006            };
1007        }
1008    }
1009}

1013/// A deserializer for a single tag item of a mixed sequence of tags and text.
1014///
1015/// This deserializer are very similar to a [`MapValueDeserializer`] (when it
1016/// processes the [`DeEvent::Start`] event). The only difference in the
1017/// [`deserialize_seq`] method. This deserializer will perform deserialization
1018/// from the textual content between start and end events, whereas the
1019/// [`MapValueDeserializer`] will iterate over tags / text within it's parent tag.
1020///
1021/// This deserializer processes items as following:
1022/// - numbers are parsed from a text content between tags using [`FromStr`]. So,
1023///   `<int>123</int>` can be deserialized into an `u32`;
1024/// - booleans converted from a text content between tags according to the XML
1025///   [specification]:
1026///   - `"true"` and `"1"` converted to `true`;
1027///   - `"false"` and `"0"` converted to `false`;
1028/// - strings returned as a text content between tags;
1029/// - characters also returned as strings. If string contain more than one character
1030///   or empty, it is responsibility of a type to return an error;
1031/// - `Option` are always deserialized as `Some` using the same deserializer,
1032///   including `<tag/>` or `<tag></tag>`;
1033/// - units (`()`) and unit structs consumes the whole element subtree;
1034/// - newtype structs forwards deserialization to the inner type using
1035///   [`SimpleTypeDeserializer`];
1036/// - sequences, tuples and tuple structs are deserialized using [`SimpleTypeDeserializer`]
1037///   (this is the difference): text content between tags is passed to
1038///   [`SimpleTypeDeserializer`];
1039/// - structs and maps are deserialized using new instance of [`ElementMapAccess`];
1040/// - enums:
1041///   - the variant name is deserialized using [`QNameDeserializer`] from the element name;
1042///   - the content is deserialized using the same deserializer:
1043///     - unit variants: consuming a subtree and return `()`;
1044///     - newtype variants forwards deserialization to the inner type using
1045///       this deserializer;
1046///     - tuple variants: call [`deserialize_tuple`] of this deserializer;
1047///     - struct variants: call [`deserialize_struct`] of this deserializer.
1048///
1049/// [`deserialize_seq`]: #method.deserialize_seq
1050/// [`FromStr`]: std::str::FromStr
1051/// [specification]: https://www.w3.org/TR/xmlschema11-2/#boolean
1052/// [`deserialize_tuple`]: #method.deserialize_tuple
1053/// [`deserialize_struct`]: #method.deserialize_struct
1054struct ElementDeserializer<'de, 'd, R, E>
1055where
1056    R: XmlRead<'de>,
1057    E: EntityResolver,
1058{
1059    start: BytesStart<'de>,
1060    de: &'d mut Deserializer<'de, R, E>,
1061}
1062
1063impl<'de, 'd, R, E> ElementDeserializer<'de, 'd, R, E>
1064where
1065    R: XmlRead<'de>,
1066    E: EntityResolver,
1067{
1068    /// Returns a next string as concatenated content of consequent [`Text`] and
1069    /// [`CData`] events, used inside [`deserialize_primitives!()`].
1070    ///
1071    /// [`Text`]: crate::events::Event::Text
1072    /// [`CData`]: crate::events::Event::CData
1073    #[inline]
1074    fn read_string(&mut self) -> Result<Cow<'de, str>, DeError> {
1075        self.de.read_text(self.start.name())
1076    }
1077}
